你现在要为我们的 MVP 做一个“核心流程可视化页面”，只做主流程展示，不要做完整官网，不要做 marketing landing page。

这次只做一个目标：

把“参考视频 → AI拆解 → 生产判断 → 创作者补拍任务 → 最终输出”
这条链路，用一个非常直观、可交互的页面讲明白。

重要要求：
- 这次不要做 Hero 区
- 不要做 Before / After 对比区
- 不要做底部品牌总结区
- 只做中间最核心的主流程展示区域
- 页面要像产品演示，不像后台表格
- 页面要适合给投资人看，一眼能懂
- 用 mock data
- 只围绕一个 talking-head / 口播视频案例展开

页面目标：
让第一次看到这个页面的人，在 15 秒内理解：
“我们把一条参考口播视频，自动拆解成多个片段，判断哪些可以 AI 生成、哪些必须真人补拍，并把真人任务压缩到最少，最后生成可交付视频。”

请你做一个单页 React demo 页面，重点只实现下面 5 个模块：

--------------------------------------------------
1. Reference Video
--------------------------------------------------

做一个参考视频输入卡片，展示：
- 视频标题
- 视频类型：Talking-head product pitch
- 总时长：28s
- 核心 message：例如 “Explain the product benefit in a fast, direct way”
- 一个 mock thumbnail / video placeholder

视觉上要让人感觉：
这是整个流程的输入源。

--------------------------------------------------
2. AI Decomposition
--------------------------------------------------

这是页面重点之一。

把参考视频拆成 5 个 segment，并清楚展示出来，例如：

- Segment 1 — Hook — 0-4s
- Segment 2 — Problem — 4-9s
- Segment 3 — Solution Demo — 9-15s
- Segment 4 — Trust / Proof — 15-22s
- Segment 5 — CTA — 22-28s

每个 segment 卡片至少展示：
- narrative role
- duration
- short script
- scene type

要求：
- 整体看起来像“视频被 AI 拆解成可执行结构”
- 可以点击某个 segment
- 点击后右侧显示 detail panel，展示：
  - full script
  - scene type
  - production mode
  - whether creator capture is needed

--------------------------------------------------
3. Production Decision
--------------------------------------------------

这是整个页面最关键的区域。

请把每个 segment 的生产判断明确展示出来，至少区分：

- AI Generated
- Creator Required

建议做法：
- 可以做成两列布局
- 或 segment 卡片上直接打明显标签
- 要让人一眼看到：
  - 5 段中哪些由 AI 完成
  - 哪些需要真人补拍
  - 真人只需要补很少一部分

案例 mock data 建议：
- 5 个 segments
- 其中 3 个 AI generated
- 2 个 creator required

同时要有一个 summary 区块，清楚显示：
- 5 segments decomposed
- 3 AI-generated
- 2 creator-required
- only 6s creator capture needed

这一块必须做得非常直观。

--------------------------------------------------
4. Creator Task Cards
--------------------------------------------------

把需要真人补拍的部分，转成任务卡片展示。

每个 task card 只展示少量高价值信息：
- task title
- which segment it belongs to
- duration needed
- framing
- action
- acceptance criteria

例如：
Task 1
- Segment: Hook
- Duration: 3s
- Framing: Mid close-up
- Action: Speak directly to camera with stronger emphasis
- Acceptance:
  - Face clearly visible
  - Quiet environment
  - Script delivered in one take

要求：
- 让投资人一眼看出：创作者不是自由发挥，而是在执行极小、极明确的拍摄任务
- 点击 task card 可以展开更多 instruction / acceptance criteria

--------------------------------------------------
5. Final Output
--------------------------------------------------

做一个最终输出卡片，展示：
- Final delivery status: Ready for review
- total duration
- AI scenes count
- Creator scenes count
- 一个 mock preview panel / timeline preview

要求：
- 让人感觉这个流程最终会产出一个可交付视频
- 不需要真实视频播放，但要有明显 preview 感
- 可以展示 timeline blocks，区分 AI 和 Creator 片段

--------------------------------------------------
交互要求
--------------------------------------------------

只做轻量交互，不要复杂拖拽：

1. 点击某个 segment
→ 右侧 detail panel 更新

2. 点击某个 creator task
→ 展开任务详情和验收标准

3. 点击 final output preview
→ 展示 mock timeline / scene breakdown

--------------------------------------------------
视觉要求
--------------------------------------------------

- 高级、克制、偏产品演示风格
- 大圆角
- 留白充足
- 轻阴影
- 不要传统 admin dashboard 密集风格
- 不要复杂表格
- AI Generated / Creator Required / Ready for Review 必须有明确视觉区分
- 风格像 early-stage startup 的 polished demo

--------------------------------------------------
数据要求
--------------------------------------------------

请直接内置一份 talking-head mock data：

- total duration: 28s
- total segments: 5
- AI generated segments: 3
- creator required segments: 2
- creator capture needed: 6s
- final status: Ready for review

示例 segment 可参考：
1. Hook
2. Problem
3. Solution Demo
4. Proof
5. CTA

--------------------------------------------------
代码要求
--------------------------------------------------

- 用 React 实现
- 单页 demo
- 组件化，但不要过度工程化
- mock data 单独抽离
- 本地可直接运行
- 页面只围绕这一个核心流程
- 不要额外扩展成完整后台系统

输出要求：
1. 先给我页面的信息架构
2. 再写代码
3. 最后告诉我如何本地运行
4. 不要只做静态展示，要有轻量交互
